為您的GitHub Pages網站配置發布源

如果您為GitHub Pages網站使用默認發布源，則您的網站將自動發布。您也可以選擇從其他分支或文件夾發佈網站。

具有存儲庫管理員或維護者權限的人員可以為GitHub Pages網站配置發布源。

GitHub Pages可在公共存儲庫中使用GitHub Free和GitHub Free用於組織，在公共和私有存儲庫中可使用GitHub Pro，GitHub Team，GitHub Enterprise Cloud和GitHub Enterprise Server。有關更多信息，請參見“ GitHub的產品”。

在本文中

  • 選擇發布來源
  • 解決GitHub Pages網站發布問題

有關發布源的更多信息，請參見“關於GitHub Pages”。

選擇發布來源

在配置發布源之前，請確保要用作發布源的分支已存在於存儲庫中。

  1. 在GitHub上，導航到站點的存儲庫。

  2. 在您的存儲庫名稱下，點擊 設置。

  3. 在“ GitHub頁面”下，使用“無”或“分支”下拉菜單，然後選擇一個發布源。

  4. （可選）使用下拉菜單為您的發布源選擇一個文件夾。

  5. 點擊保存。

解決GitHub Pages網站發布問題

注意：如果您的網站沒有自動發布，請確保具有管理員權限和經過驗證的電子郵件地址的人已推送到發布源。

如果選擇docs任何分支上的文件夾作為發布源，然後/docs從存儲庫中該分支上刪除該文件夾，則將無法構建您的站點，並且會收到頁面構建錯誤消息，提示缺少的/docs文件夾。有關更多信息，請參閱“對GitHub Pages網站的Jekyll構建錯誤進行故障排除”。
